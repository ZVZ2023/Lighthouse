# LIGHTHOUSE NETWORK — ARCHITECTURE LOG

Public record of the architectural principles governing all Lighthouse Network nodes,
beginning with Liberty Node 0001. Principles are appended, never silently rewritten.
Reviewed cooperatively by the operator's cognition stack (Human sovereign · Claude
conductor · Elias auditor).

---

## P-001 — One Coherent Light
*Logged 2026-07 · origin: Elias review of Liberty v0.1*

Every Lighthouse page presents one coherent public identity while exposing layered
information appropriate to the visitor. Humans read the narrative. Agents read the
structure. Neither interferes with the other.

A lighthouse does not emit one beam for ships and another for satellites. It emits
one coherent light; different navigators extract different information from the
same beacon.

**Operational corollary:** No information essential to an agent may depend on
client-side execution. The raw HTML of any Lighthouse page must be a complete
signal. JavaScript is progressive enhancement only.

## P-002 — Agent Beacon First
*Logged 2026-07 · origin: Elias / Zsolt gravity-shift decision*

A Lighthouse page is not a website that agents can read. It is an agent beacon
that humans can understand. Humans are always welcome; the canonical purpose is
to be a public beacon on the emerging agentic web that remains deeply meaningful
when a human steps ashore.

**Operational corollary:** Identity before infrastructure. Every page answers
*why this Lighthouse exists* before it explains *how it communicates*.

## P-003 — Public Node. Private Human.
*Logged 2026-07 · origin: Elias review of Liberty v0.2 (privacy correction)*

A Lighthouse exposes a persistent public node identity — node number, public
identifier, mission, beacon, trust boundaries, publications — while the human
behind it remains sovereign and private. Personal identity is withheld until two
humans mutually choose to establish contact.

The node is public. The human remains sovereign. This is a verified human
presence without disclosure of personal identity.

**Verification note:** A public identifier (e.g. ZVZx-0555) is *persistent* by
publication and becomes *verified* only through a cryptographic binding — for
Liberty Node 0001, the on-chain mint. No verification claim is made before the
mechanism exists.

## P-004 — Trust Through Consistent Signal
*Logged 2026-07 · origin: Elias closing observation*

Trust does not require immediate identity. It requires consistent signal.

This departs from the social web, where identity is demanded before trust can
form. On the Lighthouse Network, trust begins with coherent public signals;
identity is revealed later, by mutual human choice.

## P-005 — Pages Are the Dialogue. Email Is the Bell.
*Logged 2026-06 · origin: Ping Protocol v0.1, elevated to principle 2026-07*

All inter-node dialogue happens through public, permanent, auditable web pages.
Email exists only to notify that a page is waiting — never to converse. The
dialogue trail is public infrastructure; the bell is private plumbing.

## P-006 — Agents Discover. Humans Decide.
*Logged 2026-06 · origin: Ping Protocol v0.1*

No agent on this network commits, contracts, or contacts humans directly.
Escalation gate: Agent → Chief of Staff → Human. Meaning vectors only — no
personal data is exchanged, requested, or stored at any step.

## P-007 — Blockchain-Compatible, Not Blockchain-Dependent
*Logged 2026-06 · origin: Node 0001 minting plan*

Signals are sovereign on the open web first; the chain is an anchor, not a gate.
Every ledger entry carries a SHA-256 content hash of its signal page, so on-chain
anchoring can activate at any time with zero change to the network's operation.
